About

Cryptogram empowers users to protect their privacy when sharing photos online. By password-protecting photos with Cryptogram, users have better control over who can see photo content. Cryptogram takes regular JPEG images as input and produces images that look like random noise. With the correct password, the original images can be restored. The encoding format is designed to be robust to the levels of JPEG compression seen on social networking websites (e.g., Facebook and Google Plus). This means you can upload Cryptogram'd images to the web and the original image can be decoded using the Cryptogram browser extension and right password.

Why

What happens when users upload images to social media sites without a full understanding of the privacy settings? Images may be shared inappropriately with friends, acquaintances, or strangers. This project provides a way for people to take increased control over the privacy of personal images. Cryptogram offers password-level security inside JPEG itself. Encrypted images can be used in a variety of scenarios, including websites, social media, or e-mail. But in all cases, password protection is obtained that does not rely on the privacy settings or implementation of any particular service provider.

Technical

Image data is converted into base-64 which is then encrypted using AES block cipher, as implemented by SJCL. The resulting encrypted base-64 data is written to a JPEG with a scheme that uses 2x2 pixel blocks to represent an octal (base-8) number. Octal values are represented as shades in the luminance (Y) channel of the YCbCr colorspace. Cryptogram images can withstand JPEG compression down to approximately 72% quality (using standard libjpeg/libjpeg-turbo codecs) before the decoder fail. As JPEG quality decreases, eventually some base-8 values will become corrupted making decryption impossible without built-in error correction.

The code is open source and available on GitHub.
